## Interview Prep Notes

### React Fundamentals (Fresh — Priority)
- Custom hooks: build `useFetch`, `useDebounce`, `useLocalStorage`, `useWebSocket`
- useEffect dependency array rules, cleanup functions
- Controlled vs uncontrolled components
- Composition patterns (children prop, render props)
- `useMemo` (value) vs `useCallback` (function) — know the difference cold
- Ember-to-React mental map: Services→Context+hooks, Computed→useMemo, Observers→useEffect

### DSA Basics (For LeetCode-Present Roles)
- Arrays + two pointers, sliding window, frequency maps
- Hashmaps — counting, grouping, O(n) lookup
- Trees — BFS, DFS, recursive traversal only
- Resource: NeetCode.io — Easy + low-Medium, Arrays/Hashmaps/Trees only

### Frontend System Design Framework
1. Clarify (users, scale, devices)
2. Component tree (top-down hierarchy)
3. State (local vs lifted vs global)
4. Data flow (REST, WebSocket, polling)
5. Performance (virtualization, lazy loading, memoization)
6. Edge cases (offline, errors, loading, empty states)

### STAR Story Bank (5 Must-Haves)

- **Owned something complex end-to-end:** Ember platform feature — no docs, full ownership, data model archaeology, built cascade logic that respected individual overrides. 10-20 min workaround eliminated entirely. 80% efficiency gain. Presented to design lead and managers at completion.
- **Pushed back on a bad technical decision:** Accessibility / focus trap demo — client requested first-load site takeover, manager didn't see the problem. Built a test experience, recorded a demo showing focus lock in real time, gave manager a live test environment. Outcome: aligned on accessible solution without blocking the client request. (Evidence over argument.)
- **Navigated ambiguous requirements:** Any integration work at Wunderkind — client environments were undocumented, unpredictable, and hostile. Default operating condition.
- **Collaborated cross-functionally:** Both Ember and Chrome extension work were initiated by observing designer/engineer pain points, not from a ticket. Proactively bridged eng and design.
- **Performance or scale win:** Zero CWV degradation on high-traffic client sites under strict SLAs. Chrome extension reduced integration testing time by 70%.

- Scrimba Learn TypeScript (free, 4.2 hrs) — start here
- Total TypeScript (totaltypescript.com) — intermediate/advanced, generics, utility types
- Type Challenges (github) — 30 min/day pre-interview sharpener
